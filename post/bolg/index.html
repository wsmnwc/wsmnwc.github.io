2021年末碎碎念

Dec 28, 2021
1 minute read

入冬了在家是真的好冷，暖气聊胜于无，穿两三层还是会手脚冰凉

记录一下部署blog中遇到的一些问题

建好仓库准备push的时候遇到[fatal: The current branch master has no upstream branch. To push the current branch and set the remote as upstream, use git push –set-upstream origin master] 随后用 git push -u origin master给报了 fatal: unable to access ‘https://github.com/wsmnwc/wsmnwc.github.io.git/': gnutls_handshake() failed: The TLS connection was non-properly terminated. 看来是网络问题，于是配置代理 $ git config –global https.https://github.com.proxy https://127.0.0.1:7890 $ git config –global http.https://github.com.proxy http://127.0.0.1:7890

之后又出现 [remote: Support for password authentication was removed on August 13, 2021. Please use a personal access token instead.]的错误

大概意思就是，你原先的密码凭证从2021年8月13日开始就不能用了，必须使用个人访问令牌（personal access token），就是把你的密码替换成token 之后通过https://zhuanlan.zhihu.com/p/414028184/ 的方法

生成token后，记得把你的token保存下来，以便进行后面的操作。把token直接添加远程仓库链接中，这样就可以避免同一个仓库每次提交代码都要输入token了。

git remote set-url origin https://<your_token>@github.com//.git <your_token>：换成你自己得到的token ：是你自己github的用户名 ：是你的仓库名称 下面是一个实例： git remote add origin git://ghp_@github.com/wsmnwc/wsmnwc.github.io.git

不过由于刚才添加过一次远程github仓库，所以这次提示：fatal: remote origin already exists. 解决办法如下： 1、先删除远程 Git 仓库

$ git remote rm origin

2、再添加远程 Git 仓库

$ git remote add origin git@github.com:FBing/java-code-generator

关联成功之后，这次push又遇到[fatal: Unable to look up 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@github.com (port 9418) (Name or service not known)] 查了一下好像也是网络问题，之后用[git config –global url.“https://".insteadOf git://]替换了https，成功push

回到git仓库中可以看到网页信息已经在仓库中显示了，这时候就可以用这里这个域名来访问blog了
